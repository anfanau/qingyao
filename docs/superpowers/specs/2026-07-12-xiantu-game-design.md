# 仙途 — 前端原型完整设计规范

**日期：** 2026-07-12
**状态：** 已确认
**技术栈：** Vite + React 18 + TypeScript + Tailwind CSS 3 + Lucide React（纯 CSS 动画，零额外依赖）

---

## 一、信息架构与页面流转

```
[启动页 Splash] → [主页面 SectOrbit] → [宗门详情 SectDetail] → [游戏壳 GameShell]
                                                                    ├── 聊天 Chat
                                                                    ├── 角色 Character
                                                                    └── 功法 Skills

全局覆盖层（可从任意处触发）:
  ├── 设置 SettingsModal
  ├── 典籍/世界书 LorebookModal
  ├── 法门/预设 PresetModal
  ├── 修炼历程 ChatModal
  ├── 角色创建 CharacterCreation（首次或手动触发）
  └── Toast 通知系统
```

**关键流转逻辑：**
- 启动页 → 主页面：门开动画 → 标题展示2秒 → 自动淡出进入
- 主页面 → 宗门详情：点击轨道上任意宗门节点 → 节点平移动画 → 视图切换
- 宗门详情 → 游戏壳：点击下属宗门节点 → 进入"开局"
- 游戏壳内：左侧 sidebar 切换聊天/角色/功法三面板
- 返回路径：游戏壳 → 宗门详情（返回按钮）；宗门详情 → 主页面（返回按钮）

**状态管理：** 单层 React state（`useState` + `useReducer`），无路由库。由一个 `currentView` 枚举 + 少量上下文数据驱动全部视图切换。

## 二、启动页（SplashScreen）

### 视觉层次

```
┌─────────────────────────────────────────┐
│           深邃虚空背景（CSS 多层）          │
│   • 底层：纯黑 #050508                    │
│   • 中层：径向渐变星云（深蓝紫 + 暗金微光）  │
│   • 上层：随机星点粒子（> 60 颗，飘浮动效）  │
│                                          │
│          ┌──────────────┐                │
│          │   金色大门    │  ← 占画面 60%~65%
│          │   闭合状态    │    双扇对开结构
│          │   金色描边    │    门缝中央竖线
│          │   门钉/门环   │
│          └──────────────┘                │
│                                          │
│        「 开  门 」 ← 门下方 CTA 按钮       │
│         篆体/碑体，金色呼吸灯效              │
└─────────────────────────────────────────┘
```

### 动画序列

| 阶段 | 时长 | 描述 |
|------|------|------|
| 待机 | ∞ | 门体微弱呼吸光效（`box-shadow` 脉动 3s 周期），星空粒子漂浮，按钮呼吸灯 |
| 点击"开门" | 0ms | 按钮消散，门缝金光溢出增强 |
| 开门动画 | 1.2s | `cubic-bezier(0.87, 0, 0.13, 1)` 缓动，双扇 3D rotateY 展开（左 -110°，右 +110°），门缝溢出白光 |
| 白光绽放 | 0.3s | 门完全打开瞬间，纯白闪光覆盖全屏 |
| 标题呈现 | 0.5s | 白光消散，"仙途" SVG 标题从中心浮现 + 缩放（0.3 → 1.0 scale） |
| 标题驻留 | 2.0s | "仙途" 停留，金光在笔画上流转（SVG linearGradient 偏移动画），周围金色粒子飘散 |
| 淡出 | 0.6s | 整体 `opacity → 0`，渐入主页面 |

### 大门结构

- **门框**：深色石材纹理（CSS 多层 border + `box-shadow`）
- **门扇**：每扇宽度 ~30% 画面宽，深木色底 + 金色边框描边
- **门钉**：9行 × 9列铜色圆钉（CSS `radial-gradient` 阵列），寓意九九归一
- **铺首（门环）**：椒图兽面纹样，用纯 CSS / 内联 SVG 实现，位于门扇中央偏下
- **门缝**：闭合时微弱金光从缝中渗出

### "开门"按钮

- 位置：门正下方 40px
- 字体：`font-title`（ZCOOL XiaoWei）
- 效果：金色文字 + 外发光 + 呼吸动画（`opacity` 在 0.6~1.0 间循环）
- 点击后：文字渐变消散

### "仙途"标题（SVG 矢量）

技术方案：内联 SVG 矢量路径（北魏碑体 + 行书风格，刚劲有力、棱角分明）。

三层视觉效果：
1. **填充**：`linearGradient` 烫金渐变（`#d4a843` → `#f5d78a` → `#b8860b` → `#d4a843`），渐变位置可动画偏移实现"金光流转"
2. **描边**：`stroke="#ffffff"` `stroke-width="2"` 白色描边
3. **外发光**：`filter="url(#gold-glow)"` — `feGaussianBlur` + `feComposite` 组合实现金色外发光

尺寸：占屏幕高度 40%（`height: 40vh`），宽度等比缩放。

## 三、主页面·宗门轨道图（SectOrbitView）

### 布局

```
┌─────────────────────────────────────────────────────┐
│  [仙途] 标题                    [历程] [典籍] [法门] [设置] │ ← 顶栏
├──────────┬──────────────────────────────────────────┤
│  宗门概况 │              ┌─────────┐                 │
│  信息区   │              │  青曜    │ ← 中央轴心        │
│          │              │  (光晕)  │                 │
│  • 选中   │              └────┬────┘                 │
│    宗门   │        ╱          │          ╲           │
│    简介   │      ●宗门1   ●宗门2   ●宗门3             │
│  • 正道/  │     ╱                                ╲   │
│    邪道   │   ●宗门9    [轨道圆环]      ●宗门4        │
│  • 势力   │     ╲                                ╱   │
│    统计   │      ●宗门8   ●宗门7   ●宗门6             │
│          │        ╲          │          ╱           │
│          │              ●宗门5                       │
├──────────┴──────────────────────────────────────────┤
│              背景：动态星空（CSS 粒子）                  │
└─────────────────────────────────────────────────────┘
```

### 中央轴心 — "青曜"

- 圆形徽章，直径 ~80px
- 深色底（墨玉质感）+ 金色边框
- 内部书写"青曜"二字（`font-title`，金色）
- 外层：三层扩散光晕（`box-shadow` + `::before` + `::after`），颜色 金色→青色→淡白，持续呼吸脉动
- 光晕脉动周期 4s，`ease-in-out`

### 轨道圆环

- 半径 ~220px（响应式），以"青曜"为圆心
- 双轨道设计：内轨金色 `rgba(212,168,67,0.15)`、外轨青色 `rgba(34,211,238,0.1)`，均为虚线圆环
- 内外轨反向旋转制造视觉深度（内轨 60s 顺时针，外轨 80s 逆时针）

### 宗门节点（9个）

- 每个节点 = 圆形头像框（56px）+ 下方名称标签
- 头像框内含宗门图标（lucide-react）+ 代表色渐变背景
- 名称标签：`font-title` 小号，金色文字
- **关键行为**：节点自身不旋转——整体沿轨道公转（CSS `@keyframes orbit { 100% { transform: rotate(360deg); } }`），节点内部 `transform: rotate(-θ)` 反向抵消，始终保持正立
- 公转周期：60s 匀速

### 左侧宗门概况区

- 宽度 ~280px，半透明深色面板
- 默认显示："点击轨道上的宗门以查看详情"
- 选中宗门后展示：宗门全名 + 类型标签、宗门描述、下属宗门数量、"查看详情 →" 按钮

### 动态星空背景

- CSS `radial-gradient` 多层叠加模拟星云
- 星点（白色/淡金/淡青色，1-3px）散布画面各处，~80-120 颗
- 每颗星独立闪烁周期（通过 JS 生成随机位置和 `animation-delay`）
- 纯 CSS 渲染，零 JS 动画帧

### 微交互

| 交互 | 效果 |
|------|------|
| 悬停宗门节点 | 头像框放大 1.15x + 光晕增强 + 名称变亮 |
| 点击宗门节点 | 节点移动至中心放大，触发视图切换 |
| 悬停"青曜" | 光晕脉动加速 + 文字亮度提升 |
| 节点公转 | 匀速 60s 完整一圈，`animation: orbit 60s linear infinite` |

- 总时长 ~0.6s，`cubic-bezier(0.34, 1.56, 0.64, 1)`（带微弹性）

- 圆形头像框 140px，代表色渐变背景

## 六、角色创建（CharacterCreation）

## 七、Toast 通知系统

| 类型 | 图标 | 左边框色 | 使用场景 |
|------|------|----------|----------|
| 成功 | `CheckCircle` | 翠绿 `#10b981` | 功法突破、物品获得 |
| 信息 | `Info` | 灵青 `#22d3ee` | 天道提示、剧情推进 |
| 警告 | `AlertTriangle` | 金色 `#d4a843` | 灵力不足、境界瓶颈 |
| 错误 | `XCircle` | 朱红 `#dc2626` | 操作失败 |

### 行为

- 位置：右上角固定，距顶 80px
- 入场：右侧滑入 + 淡入，400ms，`cubic-bezier(0.16, 1, 0.3, 1)`
- 驻留：3-6 秒
- 退出：淡出 + 上移 10px，300ms
- 堆叠：最多3条
- 手动关闭 + 底部进度条倒计时

## 十、技术要点

### 动画策略

- 所有动画使用 CSS `@keyframes` + `transition`，零 JS 动画库
- 轨道公转：`transform: rotate()` GPU 加速
- 3D 门开：`transform: rotateY()` + `perspective`
- 门/节点从轨道"飞出"平移动画：动态计算偏移后用 CSS `transition` 执行

### 性能优化

- 星空粒子：CSS-only（`radial-gradient` + `box-shadow` 多层），零 JS 动画帧
- 组件懒加载：`React.lazy` + `Suspense`
- 图片零外部依赖：全部图标用 lucide-react SVG
- 字体：Google Fonts（已加载 ZCOOL XiaoWei, Ma Shan Zheng, Noto Serif SC, Inter）

### 状态管理

- 无路由库，单层 React state 驱动全部视图
- `useGameState` Hook 封装游戏全局状态（当前视图、选中宗门、激活游戏、侧边栏Tab、玩家数据）
